# Sequence Type Transitions (serial <-> GENERATED AS IDENTITY)

> pgschema issue [#279](https://github.com/pgplex/pgschema/issues/279) (closed)

## Context

PostgreSQL supports several auto-increment styles: `serial` / `bigserial`,
`GENERATED ALWAYS AS IDENTITY`, and `GENERATED BY DEFAULT AS IDENTITY`.
Converting between these styles requires different DDL patterns and careful
ownership / default handling.

pgschema originally failed to track these transitions correctly. pg-delta
used to have the same gap, and this benchmark tracked that missing parity
through pg-toolbelt issue
[#138](https://github.com/supabase/pg-toolbelt/issues/138).

Refresh note (2026-05-03): pg-delta now covers these transitions. Issue
[#138](https://github.com/supabase/pg-toolbelt/issues/138) is closed and the
fix landed in merged PR [#154](https://github.com/supabase/pg-toolbelt/pull/154).

## Reproduction SQL

```sql
CREATE SCHEMA test_schema;

CREATE TABLE test_schema.items (
    c1 int NOT NULL,
    c2 serial,
    c3 int GENERATED ALWAYS AS IDENTITY
);
```

**Change to diff:**

```sql
CREATE TABLE test_schema.items (
    c1 serial NOT NULL,
    c2 int GENERATED ALWAYS AS IDENTITY,
    c3 int GENERATED BY DEFAULT AS IDENTITY
);
```

## How pgschema handled it

pgschema expanded its column / sequence tracking so it can distinguish the
different identity modes and emit the correct transition DDL for each case.

## Current pg-delta status

| Aspect | Status |
|---|---|
| Column model captures identity metadata | ✅ `src/core/objects/table/table.model.ts` reads `a.attidentity` into `is_identity` / `is_identity_always` |
| Create / alter serializers support identity DDL | ✅ `table.create.ts` and `table.alter.ts` emit `GENERATED ... AS IDENTITY`, `SET GENERATED`, and `DROP IDENTITY` |
| Diff handles `serial <-> identity` transitions | ✅ `src/core/objects/table/table.diff.ts` compares identity flags and sequences the required operations |
| Integration regression coverage | ✅ `tests/integration/sequence-operations.test.ts` covers `GENERATED ALWAYS`, `GENERATED BY DEFAULT`, `serial and identity transition diffs`, and `identity_to_serial` |
| Existing pg-toolbelt issue / PR | ✅ [#138](https://github.com/supabase/pg-toolbelt/issues/138) closed, [#154](https://github.com/supabase/pg-toolbelt/pull/154) merged |

## Comparison of approaches

| | pgschema | pg-delta |
|---|---|---|
| **Historical root cause** | Incomplete identity / sequence transition handling | Same |
| **Current fix** | Tracks the identity mode explicitly and emits the right DDL per transition | Same |
| **Regression coverage** | Fixture-backed upstream | Integration coverage in `sequence-operations.test.ts` |

## Resolution in pg-delta

This benchmark entry is now historical. Current pg-delta extracts identity
metadata directly from the catalog, plans the necessary sequence / default /
identity operations, and covers the key transition shapes with integration
tests.
