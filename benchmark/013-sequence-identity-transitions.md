# Sequence Type Transitions (serial ↔ GENERATED AS IDENTITY)

> pgschema issue [#279](https://github.com/pgplex/pgschema/issues/279) (closed)

## Context

PostgreSQL supports both legacy serial columns and SQL-standard identity
columns. Converting between `serial`, `GENERATED ALWAYS AS IDENTITY`, and
`GENERATED BY DEFAULT AS IDENTITY` requires different DDL and sequence
ownership behavior.

pgschema fixed those transitions, and current pg-delta now covers the same
surface area. The refreshed pg-delta tree has dedicated integration tests for
identity creation and for serial/identity conversion flows in both directions.

Refresh note (2026-05-09): the original pg-toolbelt tracker
[#138](https://github.com/supabase/pg-toolbelt/issues/138) is closed, and
[pg-toolbelt#154](https://github.com/supabase/pg-toolbelt/pull/154) is merged.

## Reproduction SQL

```sql
CREATE SCHEMA test_schema;

CREATE TABLE test_schema.items (
    c1 int NOT NULL,
    c2 serial,
    c3 int GENERATED ALWAYS AS IDENTITY
);
```

**Change to diff:**

```sql
CREATE TABLE test_schema.items (
    c1 serial NOT NULL,
    c2 int GENERATED ALWAYS AS IDENTITY,
    c3 int GENERATED BY DEFAULT AS IDENTITY
);
```

## How pgschema handled it

pgschema added the missing identity-style tracking so it could emit the correct
DDL for serial-to-identity, identity-to-serial, and ALWAYS-to-BY DEFAULT
transitions.

## Comparison of approaches

| | pgschema | pg-delta |
|---|---|---|
| **Historical gap** | Identity styles were not tracked consistently | Same parity gap originally existed |
| **Current fix** | Track identity styles and emit correct transition DDL | Track identity metadata and emit the correct serial/identity transitions |
| **Regression coverage** | Upstream fixtures and tests | Dedicated integration tests for creation and both transition directions |
