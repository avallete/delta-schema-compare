# Sequence Type Transitions (serial ↔ GENERATED AS IDENTITY)

> pgschema issue [#279](https://github.com/pgplex/pgschema/issues/279) (closed)

## Context

pgschema issue #279 covered sequence-backed column transitions across legacy
`serial` defaults and SQL-standard identity columns. These transitions are easy
to get wrong because PostgreSQL uses different primitives for each direction:

- `serial` → identity requires dropping the default before adding identity
- identity → `serial` requires dropping identity, creating a sequence, and
  wiring a new `nextval(...)` default
- `GENERATED ALWAYS` ↔ `GENERATED BY DEFAULT` uses `ALTER COLUMN ... SET GENERATED`

Current pg-delta now handles all of those flows. The table model captures
identity metadata from `pg_attribute.attidentity`, `table.diff.ts` routes the
necessary `DROP DEFAULT`, `ADD GENERATED`, `SET GENERATED`, and `DROP IDENTITY`
steps, and the sequence integration suite now covers both create-time identity
columns and transition diffs.

## Reproduction SQL

```sql
CREATE SCHEMA test_schema;

CREATE TABLE test_schema.items (
    c1 int NOT NULL,
    c2 serial,
    c3 int GENERATED ALWAYS AS IDENTITY
);
```

**Change to diff:**

```sql
CREATE SEQUENCE test_schema.items_c1_seq OWNED BY test_schema.items.c1;
ALTER TABLE test_schema.items ALTER COLUMN c1 SET DEFAULT nextval('test_schema.items_c1_seq'::regclass);
ALTER TABLE test_schema.items ALTER COLUMN c2 DROP DEFAULT;
DROP SEQUENCE test_schema.items_c2_seq;
ALTER TABLE test_schema.items ALTER COLUMN c2 ADD GENERATED ALWAYS AS IDENTITY;
ALTER TABLE test_schema.items ALTER COLUMN c3 SET GENERATED BY DEFAULT;
```

## How pgschema handled it

pgschema expanded its sequence / identity tracking so it could distinguish
legacy serial defaults from identity metadata and emit the correct transition
DDL for each direction.

## Comparison of approaches

| | pgschema | pg-delta |
|---|---|---|
| **Catalog model** | Tracks serial / identity distinctions | Tracks `attidentity` state in column metadata |
| **Diff behavior** | Emits the right transition DDL for each style | Emits `DROP DEFAULT`, `ADD GENERATED`, `SET GENERATED`, and `DROP IDENTITY` as needed |
| **Regression coverage** | Fixed upstream | Integration coverage now exists for create-time identity and transition diffs |
